**测试文件**: `error-recovery.md`
**测试目标**: 验证 CCC 工作流中各阶段的错误恢复机制
**最后更新**: 2026-03-07

---

## 测试用例

### TC-ERROR-001: Intent 阶段错误恢复

**目的**: 验证 Intent 阶段的错误恢复机制

**前置条件**:
- CCC 插件已安装

**错误场景**:

| 错误 | 预期恢复 |
|------|----------|
| 空输入 | 提示用户提供需求描述 |
| 模糊需求 | 追问澄清问题或生成带假设的 Intent |
| 格式错误 | 尝试解析或提示重新输入 |

**测试步骤**:

1. 测试空输入
2. 测试模糊需求
3. 测试格式错误

**预期行为**:
```markdown
# 空输入
错误：未提供需求描述
恢复：提示用户提供
消息："请提供您想要创建的技能描述"

# 模糊需求
错误：需求不明确
恢复：生成带假设的 Intent
消息："基于以下假设生成 Intent: XXX"

# 格式错误
错误：无法解析输入
恢复：提示重新输入
消息："输入格式异常，请尝试自然语言描述"
```

**通过标准**:
- 不中断流程
- 友好的错误提示
- 提供解决建议

### TC-ERROR-002: Blueprint 阶段错误恢复

**目的**: 验证 Blueprint 阶段的错误恢复机制

**前置条件**:
- Intent 制品已生成

**错误场景**:

| 错误 | 预期恢复 |
|------|----------|
| Intent 文件不存在 | 提示检查路径或重新生成 |
| Intent 字段缺失 | 使用默认值并标注 |
| 子阶段调用失败 | 重试或跳过并标注 |

**测试步骤**:

1. 模拟 Intent 文件不存在
2. 模拟 Intent 字段缺失
3. 模拟子阶段调用失败

**预期行为**:
```markdown
# Intent 文件不存在
错误：找不到 Intent 文件
恢复：提示检查路径
消息："Intent 文件不存在：{path}。请先运行 /ccc:init"

# Intent 字段缺失
错误：缺少必填字段
恢复：使用默认值
消息："字段{field}缺失，使用默认值：{default}"

# 子阶段调用失败
错误：{stage}调用失败
恢复：重试 1 次，仍失败则跳过
消息："{stage}调用失败，跳过此阶段"
```

**通过标准**:
- 错误检测准确
- 恢复机制有效
- 标注跳过内容

### TC-ERROR-003: Delivery 阶段错误恢复

**目的**: 验证 Delivery 阶段的错误恢复机制

**前置条件**:
- Blueprint 制品已生成

**错误场景**:

| 错误 | 预期恢复 |
|------|----------|
| Blueprint 不存在 | 提示检查路径 |
| testPlan 缺失 | 使用默认测试模板 |
| 模板文件缺失 | 使用内联默认模板 |
| 文件写入失败 | 重试 1 次，内存保存 |

**测试步骤**:

1. 模拟 Blueprint 不存在
2. 模拟 testPlan 缺失
3. 模拟模板文件缺失
4. 模拟文件写入失败

**预期行为**:
```markdown
# Blueprint 不存在
错误：找不到 Blueprint 文件
恢复：提示检查路径
消息："Blueprint 文件不存在：{path}"

# testPlan 缺失
错误：缺少 testPlan 字段
恢复：使用默认测试模板
消息："testPlan 缺失，使用默认测试模板"

# 模板文件缺失
错误：模板文件不存在
恢复：使用内联默认模板
消息："模板文件缺失，使用内联默认模板"

# 文件写入失败
错误：无法写入文件
恢复：重试 1 次，内存保存
消息："写入失败，结果保存在内存中"
```

**通过标准**:
- 错误检测准确
- 恢复机制有效
- 结果可访问

### TC-ERROR-004: 完整工作流错误恢复

**目的**: 验证完整工作流的错误恢复机制

**前置条件**:
- CCC 插件已安装

**错误场景**:

| 错误 | 预期恢复 |
|------|----------|
| Intent 成功，Blueprint 失败 | 保留 Intent，提示修复 Blueprint |
| Blueprint 成功，Delivery 失败 | 保留 Blueprint，提示修复 Delivery |
| 阶段超时 | 保存进度，提供恢复命令 |

**测试步骤**:

1. 模拟 Blueprint 失败
2. 模拟 Delivery 失败
3. 模拟阶段超时

**预期行为**:
```markdown
# Blueprint 失败
状态：Intent 完成，Blueprint 失败
恢复：保留 Intent 制品
消息："Blueprint 生成失败，可使用 /ccc:design --intent-id={id} 重试"

# Delivery 失败
状态：Blueprint 完成，Delivery 失败
恢复：保留 Blueprint 制品
消息："Delivery 生成失败，可使用 /ccc:build --artifact-id={id} 重试"

# 阶段超时
状态：阶段超时
恢复：保存进度
消息："阶段超时（>{timeout}s）。已保存进度，使用 {command} 继续"
```

**通过标准**:
- 保留已完成制品
- 提供恢复命令
- 支持断点续传

## 边界测试

### TC-ERROR-Boundary-001: 连续错误处理

**目的**: 验证连续错误发生时的处理

**前置条件**:
- 模拟多个错误同时发生

**测试步骤**:

1. 模拟多个错误
2. 执行工作流
3. 观察处理

**预期行为**:
```
错误 1: 字段缺失 → 使用默认值
错误 2: 子阶段失败 → 跳过
错误 3: 写入失败 → 重试
```

**通过标准**:
- 逐个处理错误
- 不丢失错误信息
- 最终结果可用

### TC-ERROR-Boundary-002: 致命错误处理

**目的**: 验证致命错误的处理

**前置条件**:
- 模拟致命错误（如权限不足）

**测试步骤**:
