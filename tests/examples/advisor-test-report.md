# 设计顾问模式测试报告

## 测试目标
验证 advisor-core 能否通过4个问题给出正确的架构建议

## 测试用例 1: 代码审查系统

### 用户需求
"我想在每次代码提交前自动检查代码质量"

### 决策过程

**Q1: 你的主要目标是什么？**
- 回答: 事件自动化
- 推荐: Hook

**Q2: 使用场景？**
- (跳过，非Skill)

**Q3: 执行上下文？**
- 回答: 复杂任务
- 推荐: context: fork

**Q4: 复杂度？**
- 回答: 标准功能
- 预计: 3-5天完成

### 架构建议

```markdown
📋 架构建议

推荐组件: Hook + Skill 组合

架构方案:
┌─────────────┐     ┌─────────────┐
│ Pre-commit  │────▶│  Code-      │
│    Hook     │     │  Reviewer   │
│             │     │   Skill     │
└─────────────┘     └─────────────┘

配置建议:
- Hook: pre-commit 触发
- Skill: context: fork, model: sonnet
- 工具: Read, Grep, Write

实施步骤:
1. 创建 code-reviewer Skill (4小时)
2. 配置 pre-commit Hook (2小时)
3. 测试和优化 (2小时)
```

### 验证结果
✅ 建议合理，符合需求

## 测试用例 2: 文档生成工具

### 用户需求
"我需要一个工具来生成项目文档"

### 决策过程

**Q1: 你的主要目标是什么？**
- 回答: 扩展 Claude 能力
- 推荐: Skill

**Q2: 使用场景？**
- 回答: 执行具体任务
- 推荐: Task Skill

**Q3: 执行上下文？**
- 回答: 快速响应
- 推荐: context: main

**Q4: 复杂度？**
- 回答: 快速实现（MVP）
- 预计: 1-2天完成

### 架构建议

```markdown
📋 架构建议

推荐组件: Skill (Task)

配置建议:
---
name: doc-generator
description: 当用户请求生成项目文档时，读取代码并生成 Markdown 文档
context: main
model: sonnet
allowed-tools:
  - Read
  - Write
---

实施步骤:
1. 创建基础 Skill 结构 (2小时)
2. 实现文档生成逻辑 (4小时)
3. 添加示例和测试 (2小时)
```

### 验证结果
✅ 建议合理，符合需求
