# 证据链反模式 (Evidence Chain Antipatterns)

## 概述

本文档定义设计文档中证据链缺失或不完整的反模式，用于review阶段的质量检查。

## 反模式列表

### AP-EC-001: 缺少能力需求表
**严重度**: 中等
**描述**: 设计文档没有明确列出工作流各阶段需要的能力
**检测方法**: 搜索"能力需求表"或"能力需求"章节
**影响**: 无法验证设计的完整性
**修复建议**: 添加能力需求表，格式参考 docs/evidence-chain-specification.md

### AP-EC-002: 缺少Skill映射表
**严重度**: 中等
**描述**: 设计文档没有说明能力由哪些skill提供
**检测方法**: 搜索"Skill映射表"或"skill映射"章节
**影响**: 无法验证skill引用的正确性
**修复建议**: 添加Skill映射表，说明每个能力对应的skill

### AP-EC-003: Skill引用不存在
**严重度**: 高
**描述**: 映射表中引用的skill在代码库中不存在，且未标记为"新建"
**检测方法**: 对映射表中的skill路径执行文件存在性检查
**影响**: 实现时会发现缺失依赖
**修复建议**: 将不存在的skill标记为"新建"并说明创建原因

### AP-EC-004: 缺少验证清单
**严重度**: 低
**描述**: 设计文档没有提供可验证的检查点
**检测方法**: 搜索"验证清单"章节或checkbox格式
**影响**: 难以验证设计的正确性
**修复建议**: 添加验证清单，列出可检查的断言

### AP-EC-005: 数据流转不清晰
**严重度**: 中等
**描述**: 工作流各阶段的输入输出关系不明确
**检测方法**: 检查能力需求表是否包含"输入"和"输出"列
**影响**: 难以理解数据如何在阶段间流转
**修复建议**: 在能力需求表中明确标注每个阶段的输入和输出

### AP-EC-006: 能力推导缺失
**严重度**: 中等
**描述**: Skill映射表中没有说明为什么选择某个skill
**检测方法**: 检查映射表是否包含"说明"列
**影响**: 无法验证skill选择的合理性
**修复建议**: 在映射表的"说明"列中解释选择理由

### AP-EC-007: 迭代原因不明
**严重度**: 中等
**描述**: Skill状态标记为"迭代"或"新建"，但没有说明原因
**检测方法**: 检查状态为"迭代"/"新建"的行是否有详细说明
**影响**: 无法判断是否真的需要迭代或新建
**修复建议**: 说明现有skill为何不满足需求，需要如何增强

## 检查清单

审阅时必须验证以下项目：

- [ ] 包含能力需求表
- [ ] 包含Skill映射表
- [ ] 所有引用的skill存在或标记为新建
- [ ] 包含验证清单
- [ ] 数据流转路径清晰
- [ ] Skill选择有合理说明
- [ ] 迭代/新建有明确原因
