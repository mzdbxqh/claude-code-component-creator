## 🎉 重大特性：长任务持久化

CCC v3.3.0 引入了完整的长任务持久化机制，支持断点恢复和中间结果保存，解决了长时间运行任务的可靠性问题。

### 核心能力

- ✅ **断点恢复**: 从任意步骤恢复执行
- ✅ **数据持久化**: 中间结果保存到文件
- ✅ **自动检测**: 设计阶段就识别长任务
- ✅ **性能优化**: 开销 <0.02%

### 交付内容

| 类别 | 数量 | 说明 |
|------|------|------|
| 核心脚本 | 8个 | init/save/load/update/finalize/list/validate/cleanup |
| 库函数 | 2个 | common.sh, naming-rules.sh |
| 测试用例 | 45个 | 40个单元测试 + 5个集成测试 |
| 反模式规则 | 4个 | PERSIST-001 ~ PERSIST-004 |
| SubAgent | 1个 | checkpoint-validator |
| 迁移组件 | 2个 | review-aggregator, design-core |
| 文档 | 3个 | 用户指南、迁移指南、脚本文档（1387行） |

### 使用示例

**场景1：正常执行**
```bash
# 启动长任务，自动初始化事务
/cmd-review

# 系统自动保存中间结果
# 完成后标记事务完成
```

**场景2：中断恢复**
```bash
# 任务在 Step 5 中断
# 重新启动
/cmd-review

# 系统询问是否恢复
# 选择"从断点恢复"
# 从 Step 6 继续执行
```

## 🔧 技术细节

### 架构设计

```
Layer 1: 用户组件 (review-aggregator, design-core...)
         ↓ 调用持久化脚本
Layer 2: 脚本层 (scripts/persistence/*.sh)
         ↓ 读写文件
Layer 3: 存储层 (docs/{transaction-id}/, .checkpoints/)
```

### 文件组织

```
docs/{transaction-id}/{component-name}/
  ├── {key}.json                 # 配置文件
  ├── {key}-results.json         # 中间结果
  ├── {category}-results/        # 批量文件
  └── YYYY-MM-DD-{name}.md      # 最终报告

.checkpoints/{transaction-id}.json # checkpoint 元数据
```

### 性能指标

| 操作 | 延迟 | 目标 |
|------|------|------|
| init-transaction | ~50ms | <100ms |
| save-file | ~30ms | <50ms |
| load-file | ~20ms | <30ms |
| update-checkpoint | ~15ms | <20ms |

### 安全加固

- ✅ JSON 注入防护（jq -n）
- ✅ 竞态条件防护（原子操作）
- ✅ 路径遍历防护（参数验证）

## 🚀 升级指南

### 从 v3.2.0 升级

1. **更新代码**
   ```bash
   git pull origin main
   git checkout v3.3.0
   ```

2. **安装依赖**
   ```bash
   # macOS
   brew install jq

   # Linux
   sudo apt-get install jq
   ```

3. **验证安装**
   ```bash
   bash scripts/persistence/tests/test-init-transaction.sh
   ```

4. **开始使用**
   - 现有组件继续正常工作（向后兼容）
   - 新组件自动包含持久化（如果是长任务）
